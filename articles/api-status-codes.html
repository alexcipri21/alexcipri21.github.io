HTTP API Status Codes

20 Jul 2026

1. Key Distinction: 401 vs 403

Code Status Core Meaning
401 Unauthorized Authentication missing/failed. The server doesn't know who you are (missing/expired JWT token).
403 Forbidden Authorization failed. The server knows who you are, but you lack permissions (e.g. USER role accessing ADMIN endpoint).

2. 2xx Success (Request Succeeded)

  • 200 OK – Standard response for successful HTTP requests (GET, PUT, PATCH).
  • 201 Created – Resource successfully created (common response for POST requests). Includes Location header.
  • 202 Accepted – Request accepted for processing, but processing is not complete (async jobs).
  • 204 No Content – Request processed successfully, but returns no body payload (common for DELETE requests).

3. 3xx Redirection (Resource Moved)

  • 301 Moved Permanently – The requested resource URL has permanently changed.
  • 302 Found (Temporary Redirect) – Temporary URL change.
  • 304 Not Modified – Used for client-side caching. Indicates payload has not changed since last request.

4. 4xx Client Errors (Bad Client Input)

  • 400 Bad Request – Malformed request syntax, invalid JSON, or missing required body parameters.
  • 401 Unauthorized – Missing authentication or invalid credentials.
  • 403 Forbidden – Client authenticated, but lacks access rights/roles.
  • 404 Not Found – The endpoint or requested entity ID does not exist.
  • 405 Method Not Allowed – Endpoint exists, but HTTP method (e.g., POST) is disabled.
  • 409 Conflict – Conflict with current state of server (e.g., registering duplicate email).
  • 415 Unsupported Media Type – Missing or wrong Content-Type header.
  • 422 Unprocessable Entity – Syntax is correct, but payload fails validation rules (e.g. negative price).
  • 429 Too Many Requests – Rate limiting triggered (too many calls in a short window).

5. 5xx Server Errors (Server-Side Failures)

  • 500 Internal Server Error – Generic uncaught exception / crash in backend code.
  • 501 Not Implemented – Server does not support the functionality required.
  • 502 Bad Gateway – Reverse proxy (Nginx/API Gateway) received an invalid response from upstream server.
  • 503 Service Unavailable – Server is overloaded or down for maintenance.
  • 504 Gateway Timeout – Reverse proxy timed out waiting for upstream server to respond.

6. 1xx Informational (Protocol Level)

  • 100 Continue – Server received initial request headers; client should proceed to send payload body.
  • 101 Switching Protocols – Used when upgrading connection from HTTP to WebSockets.
